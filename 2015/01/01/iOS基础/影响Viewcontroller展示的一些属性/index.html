影响Viewcontroller展示的一些属性

在适配 iOS7 之前的 view 的时候，我们写过这样的代码

1 automaticallyAdjustsScrollViewInsets

描述

默认值为 YES，允许 VC 自适配 scrollview 的 contentoffset 和 contentInset 来实现 nabar 和 tabbar 的毛玻璃效果。
当同一个 VC 内有多个 scrollview 的时候，虽然值还为 YES，但是已经不起效果了，你应该将该属性设置为 NO，自己控制 scrollview 和 contentoffset 来达到你想要的结果

Default value is YES, which allows the view controller to adjust its scroll view insets in response to the screen areas consumed by the status bar, navigation bar, and toolbar or tab bar. Set to NO if you want to manage scroll view inset adjustments yourself, such as when there is more than one scroll view in the view hierarchy.

测试结果

没什么很神奇的，其实就是控制 scrollview 的 contentoffset 和 contentInset 让内容处于 nabar 和 tabbar 的之间

比如 5s 568的尺寸中，tableview.frame = self.view.bounds，系统会帮你控制的毛玻璃效果，你可以用代码写为

contentoffset

内容视图的坐标，比如 contentoffset.y = 10，说明 bounds 的坐标为 0，那 contentview 的位置就上移 10

contentInset

内边距

2 edgesForExtendedLayout

描述

指定边缘要延伸的方向，默认为 UIRectEdgeAll
在 naVC taVC 中，view 会全覆盖，意思是说 view 的视图范围会在 naBar taBar 下面，起到毛玻璃效果

This property is only applied to view controllers that are embedded in containers, such as UINavigationController or UITabBarController. View controllers set as the root view controller do not react to this property. Default value is UIRectEdgeAll.

测试结果

  • naVC.view = 320 568
  • view = 320 568
  • naVC.view = 320 568
  • view = 320 508

3 extendedLayoutIncludesOpaqueBars

描述

这个属性指定了当Bar使用了不透明图片时，视图是否延伸至Bar所在区域，默认值时NO。

Default value is NO
A Boolean value indicating whether or not the extended layout includes opaque bars.

测试结果

发现设置这个属性是无效的，因为你无论怎么设置，都是延伸的 ~
