iPhone适配Plus的一些思考（一）

~ plus 5s
屏幕尺寸 5.5 寸 736 * 414 4寸 568 * 320
宽高比例 16 * 9 16 * 9
分辨率(px) 2208 * 1242 1136 * 640
密度(density)（公式 分辨率/屏幕尺寸） 3 2
dpi 每英寸像素数 (公式（√（横向分辨率^2+纵向分辨率^2））/屏幕尺寸）) 401 326
dip 设备独立像素 dip = px / density
pt(点，point)

Android

在Android开发中多用到dip做坐标，

dip = px / density + 0.5f;
px = dip * density + 0.5f;

iOS

在iOS开发中用的就是点的概念，

~ plus 5s
分辨率 2208px * 1242px 1136px * 640px
size 736pt * 414pt 568pt * 320pt
密度 3 2

我觉得这个pt的公式也应该是

pt = px / density;
px = pt * density;

(只不过iOS现在的密度都是整数的，我们不需要加 0.5 来取整)

疑问

我们通过 Auto Layout 或者 Size Classes 中的相对坐标是否真的能解决适配

因为我们看到


这是一张设计图，针对中间元素，我们可以设 layout

  1. top = 138/2.0 pt,
  2. width == height = 234/2.0 pt
  3. 居中

但这是针对 5s 568pt 320pt 的设计，我们在 736pt 414pt 中同样使用这份 layout，你会发现，整个元素，距离上太近了，而元素又太小了，我们是不是应该同比例放大？

公式 = 设计稿中宽的 pt / 当前屏幕的 pt * 设计稿中元素的 pt

以实现等比方法 ..

方法有二

  1. 计算出比例，改变初始化 frame
  1. 按照比例放大元素

这样布局问题就解决了，因为你所有的元素都是等比放大，你的间距也是等比放大，你的 ui 基本就是一致的没有调动

但是

  1. 可以看到 storyboard 不能很好的实现这样的运算
  2. 元素的size是动态放大的，切的 @2x @3x 图可能就没有这么标准了 ..
